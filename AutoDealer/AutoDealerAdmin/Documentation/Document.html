AutoDealer System

This document is detail the specifications for the various parts of the AutoDealer System. The AutoDealer system is an admin program and a Web Site. The admin program is used to define aspects of the web site and to define the vehicles currently for sale. The web site is made up of dymanic web pages to display the data defined by the admin program.

This document has some use in using the admin program and understanding how that relates to the web site pages. It's major use is as a system design document with information intended for the developers of the AutoDealer System.

The document is broken up into 2 chapters, the Admin program and the second chapter is the Web Site.

Auto Dealer Admin

This system, because the size is small, uses a paradign where only the Form1 code can have access to MySQL. As forms are instantiated and shown, any data necessary must be passed in a form property. On return from the form the DialogResult is checked and if the OK button (Usually titled Add/Update) then the data is pulled from the forms properties and the MySQL INSERT or UPDATE is called.

This also rquires the forms to make sure the updated fields on the form have been saved in the properties.

Auto Dealer Tables

To dynamically build the web site pages information needs to be pulled from various tables. Auto Dealer uses the MySQL system to build, access and maintain these tables.

Vehicle Table

This is the table to holds the information on each vehicle that has, is or will be shortly listed on the web site. Each vehicle is assigned an ID by the system when you create a new vehicle record in the Vehicles Tab. The data that is supplied by the user is then stored in this table.

The start and end dates that this vehicle will be listed on the web site are set here. For the vehicle to be displayed the current date MUST fall between these dates.

The descriptive text that will displayed on the web site are specified in the PageDescription and ShortDescription.

The pictures that will be displayed are specified in the SmallPic and LargePic. The small pic is displayed on the vehicle detail page and in the category page. The large pic is displayed when the small picture is clicked on the vehicle detail page. The gallery pictures, if any, are specified in the gallery table. The smallPicUpload and largePicUpload are set to "1" when the user wants the picture files uploaded.

The active field can be set to 0 to "soft delete" the record.

Category Table

The category table defines the classes that a vehicle can belong to. At this time there can only be one class for each vehicle. These records are also used to fill the "Category" menu item.

You can control the sequence that the categories will be displayed in the menu by setting the seq field. When these are read in there is an ORDER BY on the seq field. Two entries can have the same seq value, and the entries will be sorted in respect to the other entries, but the order within entries with the same seq value can not be pre-determined.

The Description field is for the person entering the data in the Admin routines Category Tab. The PageDescription is optional and if specified is diplayed on the Category Page prior to any of the vehicle pictures.

Vehicle Tab

PURPOSE:The intent of the Vehicle Tab is to display ALL "ACTIVE" vehicles and to let the user select vehicles to EDIT or to ADD new vehicles. EDITING and ADDING of vehicles is accomplished by launching the frmVehicle form.

ALL information needed to populate the frmVehicle form is contained in the grid. Fields that are not necessary to be viewed have been set in display=false. When the user selects to ADD or EDIT a vehicle the frmVehicle form is created and then the properties of that form are filled in, the Gallery photos are obtained and passed as a DataTable in the ??? property and then the form is shown.

On return from the frmVehicle form IF the DialogResult is "OK" then the information about the vehicle is obtained from the properties and a vehicle record is either INSERTED or UPDATED. We take the cheap way out and refresh the grid by doing a new SELECT.

Vehicles are made "INACTIVE" by setting the Ending Date to midnight of today's date.

More information can be found at VehicleForm and VehicleTable.

Vehicle Form

PURPOSE: This form is to display the details about a vehice (if this is an EDIT or DELETE) and to let the user add or change values. In the case of an ADD the user will see empty fields and will fill-in whatever data they have. The "Name" and category fields are required.

The Vehicle section is were basic data about the vehicle is entered. Vehicle Name, Category, VIN and price.

The "Short Description" is the text that will be displayed at ??? on the web site.

The "Page Text" is displayed on the Vehile page.

This is also where the "Small" picture and "Large" picture are Selected. The "Small" picture is displayed on the Web Site Category page and on the Vehicle Page. The "Large" picture is displayed when the user clicks the "Small" picture. See "Photo Tab" for details on getting pictures into the system.

Pressing the "..." button to the right of the fileName will present you with the typical Windows File Selection form. Select the file you want for the "Small" or "Large". Once selected the picture will be displayed below. NOTE: These displays are NOT the same size as will be shown on the Web Pages.

Every attempt is made to make sure the Upload check boxes are set correctly (checked when the program thinks the picture file has NOT been uploaded yet. However here is where you can override and tell the system to upload the file or not.

The Gallery section is where you can see the Gallery pictures that have been assigned to this Vehicle. The Gallery is a set of thumb nail pictures that are displayed on the Vehicle Web page and the user can click on them to see the larger picture.

Right Clicking in the list of file names will let you select more Gallery pictures to be added or removed. The Photo page is where you can select the "Large" picture to associate with the thumb nail Gallery picture and set the Upload check box.

More information can be found at Vehicle Tab, Vehicle Table, the Photo Tab, the Vehicle Web Page and the Category Web Page.

Category Form

PURPOSE: To let the user define the Categories that will be used in the Web Site. The user can change the sort order (sequence that the Categories will be displayed) and the text that will be displayed on the Web Site.

When the Category Tab page needs to start up the Category Form, the form is instantiated, the properties are filled in from the datagrid on the Tab form and then the form is shown.

The Category name is the name that will appear in the menu on the Web Site as well as at the top of the Category Pages.

The Category Description will Display on the Category Pages on the Web Site.

The Sequence is to set the sort order. There is no need for these number to be consecutive (holes like 1,2,5,6,20) are fine. Even duplicate sequence number are accepted but you will not be able to count on the exact ordering for the items with duplicates.

Not sure what the Web Page Description will be used for at this time.

Pressing the "Add" or "Edit" buttons will cause the data to be saved and the form to be removed. The Tab page will be refreshed to show the changes/additions you have made.

When control is returned to the Category Tab, the DialogResult is checked for OK (the Add/Update button pressed). If so the code then pulls the data from the properties and makes the MySQL call to INSERT or UPDATE the data.
